## Black Box- /White Box-Tests
#### Black Box
Black box test sind tests von Personen die nicht wissen wie das Programm intern aussieht. Dies ist ideal für wenn ein Programm auf das Oberflächliche, theoretisch auf Nutzer ebene, getestet werden soll.
#### White Box
White box tests sind das genaue Gegenteil von Black-Box tests. Das sind tests die von Entwicklern geführt werden die wissen wie das Programm intern aussehen bzw. die Zugriff auf das äußere des Programmes haben.
## Testen
#### Print-Debugging
Print-Debugging beschreibt den Prozess stichproben von Variabeln zu bekommen indem man sie per Print-Befehl in die Konsole ausgibt. Dieser weg zu Debuggen ist nicht sonderlich gut zu machen, da es den Debugger gibt der essenziell das genau gleiche wie ein Print-Statement macht nur das du das Programm an der Stelle freezed.
#### TDD (Test Driven Development)
TDD beschreibt eine Arbeitsweise Tests zu schreiben bevor die Entwicklung überhaupt beginnt.
#### Unit-Test
Unit tests sind tests die Einzelne Komponenten eines Systemes aggressiv testen. Ein Beispiel hierfür wäre ein Skript welches die gleiche Funktion mit mehreren verschiedenen Attributen laufen möchte um zu sehen wie das Programm reagiert und was es dabei macht.
#### E2E-Test
Beim end-to-end testing werden Programmabläufe von Anfang (Frontend) ins Backend bis es wieder zurück an das Frontend gelangt getestet um ein einheitlichen und umfangreiche Test-umgebung geschaffen wird.
