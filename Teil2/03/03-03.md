## SQL (Strucutured Query Language)
Eine Programmiersprache die dazu dient eine Datenbank anzusprechen.
Wichtige Funktionen hier: 
```
-- Hole bestimmte, oder alle, werte aus einer Tabelle
SELECT * FROM Tabelle
```

```
-- oder
SELECT SpalteName1, SpalteName2 FROM Tabelle
```

```
-- Wenn du konditionen hinzufügen willst.
SELECT * FROM Tabelle WHERE SpalteName1 == "Hello World"
```

```
Es ist auch möglich eine Tabelle einen Variabelnamen zu geben. Welche für Inserts auch nützlich sind.
SELECT tab.SpalteName1
FROM Tabelle tab
```
Ab hier würde ich auch empfehlen das SELECT und FROM in zwei verschiedenen zeilen zu haben um es lesbarer zu halten.
```
Es ist auch möglich zwei tabellen miteinander zu verknüpfen mit Inserts.
SELECT tab.SpalteName2, tab2.SpalteName2
FROM Tabelle tab
INNER JOIN Tabelle2 tab2 ON tab.SpalteName1 == tab2.SpalteName1
```

Wichtig hier zu sagen das mann eine Hilfestellung von der IHK erhaltet. Deswegen ist es hier nur so brief. Guckt euch lediglich die Syntax an bitte.

## API
Anwendungsschnittstellen sind punkte deines Programmes die verwendet werden können um andere Programme anknüpfen zu lassen. Hierbei ist es wichtig zu notieren das so eine Anknüpfung meist über eine HTTP aufruf entsteht und dann noch über die gesetzen HTTP-Methoden wie GET, PUT, DELETE, PATCH und UPDATE.

Es gibt verschiedene Arten von API's die Beliebteste wäre eine Rest-API

## Ressourcen
Das Wort ressourcen wird oft mit dem Zusammenhang zu APIs verwendet. Grundsätzlich bedeuten die nur die Einzelnen Entitäten auf den die API zugreift. 

Zum Beispiel der Datenbank Eintrag des Nutzers 
## REST (Representational State Transfer)
REST ist grundsätzlich nur ein Regelwerk welches klar definiert wie Geräte miteinander über ein einfache HTTP-Anfrage Daten abrufen und ändern können. REST-API's verwenden das Prinzip von Ressourcen indem es eine Darstellung der geholten Ressource zurückschickt.

## SOAP
SOAP, im gegensatz zu REST-API, verwendet nur XML und ist generell viel strenger bei dem Einbau. Jedoch bietet SOAP dafür deutlich mehr einstellungsmöglichkeiten wenn zur Ansprache ist da SOAP-API's nicht auf HTTP-Anfragen (SMTP, TCP gehen auch.) begrenzt sind. Im gegensatz zu rest api sind Funktionsorientiert und nicht ressourcen orientiert.

## GraphQL
GraphQL ist eine Datenbankorientierte API art welche per JSON ähnlichaussehenden GraphQL Queries direkte abfragen an einer Datenbank ausführen können.
