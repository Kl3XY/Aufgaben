#### IPv4
IPv4 ist der veraltete dennoch am häufigsten benutzte standard in dem Netzwerk teil der Informationstechnik. Diese besteht aus vier Oktette die jeweils darstellt was für eine Art IP-Adresse es ist und in welchen Subnetz sie befindet. Anhand einer Subnetzmaske wird festgelegt welcher teil der Adresse vom Host vergeben wird und welche den einzelnen Clients zugeordnet.

Subnetzmaske sind identisch aufgebaut wie normale IPv4 Adressen jedoch sind diese keine wirklich adressen mit welcher man sich verbindet sondern eher eine Vorgabe des Netzwerks welche Teile der Adresse zu Clients vergeben werden können. Dies wird gemacht um Netzwerke in einzelne Subnetze zu unterteilen. 

Eine normale öffentliche IP-Adresse würde ungefähr so aussehen
```
192.168.25.10
```

Mit einer Subnetzmaske die ungefähr so aussieht:
```
255.255.255.0
```

Um den genauen nutzen der Subnetzmaske klar zu machen müssen die jeweiligen IP-Adressen in Bits umgewandelt werden.
```
Host-Adresse
Dezimal : 192.168.25.10
Binär: 11000000.10101000.00011001.00001010

Binär-Adresse
Dezimal: 255.255.255.0
Binär: 11111111.11111111.11111111.00000000
```

Die Subnetzmaske definiert welcher teil der Host-Adresse sich innerhalb eines Subnetzes verstellen darf. Bei einer eins in einer Subnetzmaske bleibt der wert in der Host-Adresse starr und darf nicht verändert werden. Wenn dieser verändert wird, befindet sich die Adresse in einem andern Subnetz.

## ARP
Das Address Resolution Protocol (ARP) ist ein Netzwerkprotokoll was verwendet wird um Mac-Adressen an IP-Adressen zu heften und die Kommunikation über das Netzwerk zu verbessern.
Diese Mac-Address zu IP-Adressen Zuordnung werden in eine sogenannte "ARP-Tabelle" eingespeichert.

Dieses Protokoll wird nicht für IPv6 verwendet. Hierbei wird das "Neighbour Discovery Protocol" (NDP) verwendet.
