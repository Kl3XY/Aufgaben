## Pseudocode
Das Grundprinzip von Pseudocode ist es eine grundlegende idee der Umsetzung von einer Funktion darzustellen. Pseudocode verläuft nicht einer typischen Programmiersprachen und verwendet stattdessen.

aus einem code der die Funktion "ReturnHelloWorld" repräsentieren soll wird:

> [!NOTE]- C# Snippet
> ```c#
> public string ReturnHelloWorld() {
> 	return "Hello World"
> }
> ```

wird:
>[!NOTE]- Pseudocode
>```
>funktion string ReturnHelloWorld()
>begin
>	return "HelloWorld"
>end
>```

Hierbei ist zu beachten das dies von Prüfer zu Prüfer anders sein kann. Stursberg tendiert auch normales C# zu akzeptieren... bisauf sehr extravagante syntax wie LINQ usw.

#### Klassendiagramm
Das Klassendiagramm ist das komplizierteste von den dreien aber auch zu gleich das was man am meisten sieht.

das Klassendiagramm stellt die Beziehungen zwischen den jeweiligen Klassen dar.
Ungefähr so:
![[Pasted image 20251114101710.png]]
Das hier ist die simpelste darstellung einer Klasse. Dieses Fenster representiert die Klasse ohne irgendwelche Attribute oder Methoden

Hier ist ein Beispiel mit den jeweiligen Attributen sowie auch weiteren wichtigen elementen
(Schlüsselwörter und Eigenschaften sind optional und ideal fürs einschleimen)
![[Pasted image 20251114101957.png]]

Das schwerste an dem Klassendiagramm sind die Beziehungen der jeweiligen Klassen. bzw dessen Pfeile.

**Komposition (Voller Pfeil):** Beide Teile der Beziehung benötigen einander um zu existieren.
**Aggregation (Umrandeter Pfeil):** Nur ein Teil benötigt das andere

genau welche pfeile verwendet werden müssen ist am einfachsten an der Notation zu erkennen.

(>=1)* zu (>=1) ist immer eine Komposition
(>=0) zu (>=1) ist eine Aggregation

*(größer, oder gleich, als 1)
