## Debugging, Breakpoint
Debugging beschreibt den Prozess den Code auf potenziellen Fehlern zu analysieren.

"Breakpoints" sind ein großer Hilfsmittel dieses Analyseverfahrens. Beim setzen eines Breakpoints wird das Programm an exakt diesem Punkt gestoppt. Während diesem Prozess kann der Entwickler auf die derzeitigen Variabeln und deren Werten zugreifen und sogar manipulieren.

## Software-Test
#### Black Box
Der Black Box test wird meistens von externen durchgeführt. Hierbei wird das Programm geprüft aus der Perspektive eines Nutzers der das Programm von innen nicht kennt.

#### White Box
Der White Box test ist das Gegenteil des Blackbox tests. Hierbei wird das Programm getestet mit dem Basiswissens der Innereien des Programmes.

#### Code Review
Bei dem Code Review wird von mehreren Entwicklenr der Code angeguckt und nochmals überarbeitet wenn dieses den Code Richtlinien der Firma nicht passt.

## Testdaten
Mithilfe von Testdaten Fehler finden und die Limitationen der Jetzigen implementation eines Programmes ermitteln. 


## Unit test / Komponententest
Unit tests sind automatisierte tests wo eine Funktion bzw. "Unit" eines Programms auf extremer weiße getestet wird.

## Integrationstest
Integrationstests werden durchgeführt nachdem eine Funktion den Unit Test bestanden hat. Hierbei wird die neue Funktion so getestet ob sie mit dem restlichen teil der Software stabil funktioniert.
## Systemtest
Das Gesamte system wird auf alle Funktionalen und nicht-funktionalen Anforderungen geprüft.
#### Funktionale Anforderungen
Features die das Programm oder was auch immer actually machen soll. Z.b. Ein Datenbank server ermöglicht es remote auf eine Datenbank zuzugreifen von allen standorten
#### Nicht-funktionale Anforderungen
Nummern, Prozente, du nennst es. Was für Leistungen soll das Produkt machen? z.B. die neuen Festplatten für den Server verschnellern den Speicherprozess um 49%.
#### Merksatz
Funktionale Anforderungen sind _*was*_ das System machen soll
Nicht-funktional Anforderungen sind _*wie gut*_ das System es machen soll.
## Versionsmanagement des Quellcodes
Versionsmanagementsoftware (kurz VMS) sind Dienste die dafür zuständig sind die Verschiedenen Versionen eines Programmes beizubehalten und zu Versionieren.
==Versionieren bedeutet, verschiedene Versionen oder Entwürfe von Dokumenten, Software oder anderen Dateien systematisch zu verwalten und zu speichern==

VMS Dienste basieren auf sogenannte Repositories die den Aktuellen Quellcode eines Programmes beinhalten sowie auch den Veralteten Versionen des Quellcodes. In den meisten fällen haben VMS systeme jeweils zwei Repositories mit denen sie arbeiten. Ein Lokales Repository was ich in dem Projektordner des Nutzers befindet und einem Remote Repository dessen inhalt mit dem Lokalen oft zusammengeführt wird.
